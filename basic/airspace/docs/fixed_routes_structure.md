# 固定航线库存储结构设计说明

## 1. 概述
固定航线库是为无人机仿真想定编辑系统设计的标准航线数据存储格式,支持快速引用和复用预设航线。

## 2. 存储结构说明

### 2.1 根结构
```json
{
  "航线库": {
    "元数据": {...},
    "任务类型定义": {...},
    "航线数据": [...]
  }
}
```

### 2.2 元数据 (metadata)
- `版本`: 数据格式版本号
- `创建时间`: 数据库创建日期
- `坐标系`: 使用的坐标系统(目前为WGS84)
- `高度基准`: 高度参考标准(海拔高度)

### 2.3 任务类型定义 (mission_types)
定义系统支持的任务类型及其说明:
- **物资运输**: 无人机物资配送任务,连接主要仓储点和配送站
- **应急医疗**: 紧急医疗救援航线,连接医院与紧急集合点
- **农林植保**: 农业喷洒和林业巡查航线,覆盖农田和林区
- **巡逻安防**: 边境巡逻和城市安防巡逻航线
- **应急救援**: 自然灾害救援航线,连接应急物资储备点
- **测绘巡检**: 地理测绘和基础设施巡检航线

### 2.4 航线数据结构
每条航线包含以下字段:

#### 基本属性
- `航线ID`: 唯一标识符 (格式: FIX + 三位数字)
- `航线名称`: 航线的描述性名称
- `任务类型`: 归属的任务类别
- `优先级`: 航线重要性级别 (S-最高, A-高, B-中, C-低)
- `总里程`: 航线总长度(km)
- `预计用时`: 预计飞行时长(min)
- `飞行高度`: 标准飞行高度(m)
- `航路半宽`: 航路宽度的一半(m)
- `最大速度`: 建议最大飞行速度(km/h)
- `航路级别`: 航路分级 (Ⅰ-主干, Ⅱ-支线, Ⅲ-末端)
- `备注`: 补充说明信息

#### 航路点信息
- `起点`: 包含名称、坐标[经度,纬度,高度]、类型
- `终点`: 包含名称、坐标[经度,纬度,高度]、类型
- `航路点`: 数组,包含所有中间航路点
  - `序号`: 航路点顺序号
  - `名称`: 航路点标识
  - `坐标`: [经度,纬度,高度]
  - `停留时间`: 在该点停留的时间(秒)

#### CZML数据
- `CZML`: Cesium可视化格式字符串,可直接用于3D地图渲染

## 3. 仿真想定编辑应用

### 3.1 引用机制
仿真想定编辑时,可通过以下方式引用固定航线:

```javascript
// 通过航线ID引用
const route = routeLibrary.find(r => r.航线ID === 'FIX001');

// 通过任务类型筛选
const transportRoutes = routeLibrary.filter(r => r.任务类型 === '物资运输');

// 通过优先级筛选
const priorityRoutes = routeLibrary.filter(r => r.优先级 === 'S');
```

### 3.2 想定配置示例
```json
{
  "想定ID": "SCENARIO001",
  "想定名称": "城市物资配送演练",
  "航线引用": [
    {
      "航线ID": "FIX001",
      "执行次数": 5,
      "执行间隔": 30,
      "任务时间窗": ["08:00", "18:00"]
    },
    {
      "航线ID": "FIX008",
      "执行次数": 10,
      "执行间隔": 15,
      "任务时间窗": ["08:00", "20:00"]
    }
  ]
}
```

### 3.3 动态调整参数
在引用固定航线后,可在想定中覆盖以下参数:
- `飞行高度`: 根据实际需求调整
- `最大速度`: 根据无人机性能调整
- `执行间隔`: 修改任务执行频率
- `任务时间窗`: 设置有效执行时间段

## 4. 数据特点

### 4.1 地理覆盖
- 以德清县为中心
- 覆盖周边湖州、杭州、余杭、安吉等地区
- 包含城市、山区、湖泊、森林等多样地形

### 4.2 任务多样性
- 6大任务类型
- 20条预设航线
- 覆盖日常、应急、巡检等多种场景

### 4.3 航线级别
- **Ⅰ级**: 主干航线,跨区域长距离
- **Ⅱ级**: 支线航线,区域连接
- **Ⅲ级**: 末端航线,精细化作业

### 4.4 优先级分级
- **S级**: 应急专用,最高优先级
- **A级**: 重要任务,高优先级
- **B级**: 常规任务,中优先级
- **C级**: 一般任务,低优先级

## 5. 扩展建议

### 5.1 新增航线
遵循现有结构,添加新的航线记录,确保:
- 航线ID唯一性
- 坐标精度一致性
- CZML格式正确性

### 5.2 版本管理
- 修改元数据版本号
- 记录修改日期和变更内容
- 保留历史版本以便回滚

### 5.3 数据验证
- 验证坐标在有效范围内
- 验证航路点序列合理性
- 验证CZML格式正确性

## 6. 技术实现

### 6.1 文件位置
```
c:/mgs/basic/airspace/data/airroute/fixed_routes_library.json
```

### 6.2 加载示例
```javascript
// 加载航线库
async function loadRouteLibrary() {
  const response = await fetch('./data/airroute/fixed_routes_library.json');
  const data = await response.json();
  return data.航线库.航线数据;
}

// 使用航线库
const routes = await loadRouteLibrary();
routes.forEach(route => {
  console.log(`${route.航线名称} - ${route.任务类型}`);
});
```

### 6.3 Cesium渲染示例
```javascript
// 渲染航线
routes.forEach(route => {
  const czml = JSON.parse(route.CZML);
  viewer.entities.add(czml);
});
```
